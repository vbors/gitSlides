Workflow Commands - Reset

git reset - The first thing reset will do is move what HEAD points to. Unlike checkout it does not move what branch HEAD points to, it directly changes the SHA of the reference itself.

This means if HEAD is pointing to the 'master' branch, running git reset 9e5e6a4 will first of all make 'master' point to 9e5e6a4 before it does anything else.

git reset --soft HEAD~ - it are moving the branch back to where it was without changing the Index (staging area) or Working Directory. You could now do a bit more work and commit again. like svn revert

HEAD~ - the parent revision of HEAD

git reset --mixed HEAD~ - to update the Index with the contents of whatever tree HEAD now points to.

It still undid your last commit, but also unstaged everything. You rolled back to before you ran all your git adds AND git commit.

git reset --hard HEAD~ - make the Working Directory look like the Index

It undid your last commit, all the git adds, and all the work you did in your working directory.
