The Three States Workflow Commands

The HEAD in Git is the pointer to the current branch reference, which is in turn a pointer to the last commit you made or the last commit that was checked out into your working directory.
git init - create a Git repository with a HEAD reference that points to an unborn branch (aka, nothing)
git add - take content in your Working Directory and populate our Index with the updated content
git commit - take what the Index looks like now and save it as a permanent snapshot pointed to by a commit, which HEAD is then updated to point at.
Edit -> git add -> git commit
