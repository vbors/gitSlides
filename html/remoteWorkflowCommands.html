Workflow Commands - Remote Repository

git clone - Clone a repository into a new directory

Clones a repository into a newly created directory, creates remote-tracking branches for each branch in the cloned repository , and creates and checks out an initial branch that is forked from the cloned repositoryâ€™s currently active branch.

We clone an existing repository with command git clone [remote_repo] where [remote_repo] is the repository link.
Example: git clone username@host:/path/to/repository, this will checkout the active branch master.
git clone -b [branch] [remote_repo] Clone only one remote branch, where -b will move HEAD to the new local branch
git fetch - Update local repository from remote
git pull - Fetch from and integrate with another repository or a local branch = git fetch; git merge FETCH_HEAD
git push - Updates remote refs using local refs, while sending objects necessary to complete the given refs.

git push origin [branchname]

git branch - list all branches. -r flag will filter only by remote branches
git branch -b [branchname] - Create new local branch, referencing the same point in history as the current branch.
git branch -d [branchname] - Delete local branch
git merge [branchname] - Merge another branch [branchname] into your active branch (HEAD)
git tag jump-2.8.0 1b2e1d63ff - Create a new tag named jump-2.8.0 the 1b2e1d63ff stands for the first 10 characters of the commit id you want to reference with your tag.
